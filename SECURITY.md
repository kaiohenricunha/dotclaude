# Security Policy

## Supported Versions

| Version | Supported          |
| ------- | ------------------ |
| 0.2.x   | :white_check_mark: |
| < 0.2   | :x:                |

## Reporting a Vulnerability

If you discover a security vulnerability in this project, please report it responsibly.

**Do NOT open a public GitHub issue for security vulnerabilities.**

Instead, please use one of these methods:

1. **GitHub Private Vulnerability Reporting** — Use the "Report a vulnerability" button on the [Security Advisories](https://github.com/kaiohenricunha/dotclaude/security/advisories) page.
2. 2. **Email** — Contact the maintainer directly at the email listed on the GitHub profile.
  
   3. ### What to include
  
   4. - Description of the vulnerability
      - - Steps to reproduce
        - - Potential impact
          - - Suggested fix (if any)
           
            - ### Response timeline
           
            - - **Acknowledgment**: Within 48 hours
              - - **Initial assessment**: Within 1 week
                - - **Fix or mitigation**: Best effort, typically within 2 weeks for critical issues
                 
                  - ### Scope
                 
                  - This policy covers:
                  - - The `@kaiohenricunha/harness` npm package (everything under `plugins/harness/`)
                    - - Shell scripts (`bootstrap.sh`, `sync.sh`)
                      - - CI workflow templates generated by `harness-init`
                        - - The settings validator (`validate-settings.sh`)
                         
                          - Personal dotfiles (`CLAUDE.md`, `commands/`, `skills/`) are provided as-is and are not in scope for security fixes.
